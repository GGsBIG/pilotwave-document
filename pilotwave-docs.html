Cluster Management

Comprehensive Kubernetes Cluster Management with Pilotwave+

Pilotwave+ provides a unified interface for managing Kubernetes clusters, offering comprehensive visibility and control over your infrastructure. The platform integrates seamlessly with your existing Kubernetes environment to deliver powerful management capabilities.

Architecture Overview

The following diagram illustrates the overall design architecture of Pilotwave+ and how it integrates with your Kubernetes clusters:

Pilotwave+ consists of three main components:

  • Frontend - User interface for cluster management and visualization
  • Backend - Core processing engine that handles cluster operations
  • Kubernetes SDK - Direct integration with Kubernetes APIs for real-time cluster interaction

This architecture enables Pilotwave+ to connect to multiple Kubernetes clusters simultaneously, providing centralized management capabilities across your entire infrastructure.

Cluster Resource Management

Pilotwave+ offers detailed visibility into all aspects of your Kubernetes clusters. The platform provides comprehensive management interfaces for nodes, namespaces, and other critical cluster resources.

Node Management

Monitor and manage your cluster nodes with real-time status information, resource utilization, and health monitoring:

The nodes interface displays:

  • Node Status - Real-time health and readiness status
  • Role Information - Control plane and worker node identification
  • Kubelet Version - Version tracking for consistency management
  • Internal IP - Network configuration details
  • Age - Node lifecycle tracking

You can click on any node to view detailed information including resource consumption, running pods, system services, and configuration details.

Namespace Management

Organize and manage cluster resources through namespace isolation and resource management:

The namespace management interface provides:

  • Namespace Status - Active status monitoring for all namespaces
  • Resource Isolation - Clear separation of workloads and resources
  • Label Management - Comprehensive labeling system for organization
  • Age Tracking - Namespace lifecycle information
  • Quick Actions - Direct access to namespace operations

Each namespace can be clicked to drill down into specific resources, including deployments, services, configmaps, secrets, and other Kubernetes objects within that namespace scope.

Interactive Resource Exploration

Pilotwave+ enables interactive exploration of cluster resources. You can click on any resource to view detailed information, including:

  • Resource specifications and configurations
  • Real-time status and health information
  • Event logs and troubleshooting data
  • Related resources and dependencies
  • Performance metrics and resource usage

This interactive approach makes it easy to navigate complex cluster hierarchies and quickly identify issues or optimization opportunities.

Key Features

  • Multi-Cluster Support - Manage multiple Kubernetes clusters from a single interface
  • Real-Time Monitoring - Live updates of cluster status and resource health
  • Resource Discovery - Automatic detection and categorization of cluster resources
  • Interactive Navigation - Click-through exploration of resource relationships

  • Intuitive Interface - User-friendly design for efficient cluster management

Common Questions and Answers About Pilotwave+ Kubernetes Management Platform

Find answers to the most frequently asked questions about Pilotwave+ features, deployment, configuration, and support. This comprehensive FAQ covers technical details, compatibility, security, and operational considerations.

1. What LLM providers does the AI Assistant System support? Can I use my own self-hosted LLM?

▼

Answer: Yes, Pilotwave+ supports multiple LLM providers including OpenAI, Google (Gemini), and Ollama (local deployment). The system is designed to be extensible, with future support planned for additional providers including LangGraph and other enterprise LLM solutions. You can absolutely use your own self-hosted LLM infrastructure through the Ollama integration or by configuring custom endpoints in the AI settings.

2. Is all the information displayed on the interface real-time data?

▼

Answer: Yes, absolutely. All data displayed in Pilotwave+ is retrieved in real-time directly from the Kubernetes API. This includes resource status, metrics, configurations, and system health information. The platform maintains live connections to your cluster to ensure you always see the most current state of your infrastructure.

3. What should I consider when migrating from the legacy version?

▼

Answer: Please refer to our comprehensive Migration page for detailed guidance. The key points are: all existing functionality is preserved and enhanced in Pilotwave+, both versions can coexist in the same cluster during migration, and the transition can be performed with zero downtime. We provide complete migration tools and documentation to ensure a smooth transition.

4. How do I configure different authentication methods?

▼

Answer: Authentication methods can be configured through the installation interface. Pilotwave+ supports three authentication modes: No Authentication (for internal clusters), Basic Authentication (single account with username/password), and LDAP Authentication (enterprise directory integration). You can modify these settings during initial installation or reconfigure them later through the Extensions > Auth Mode section.

5. Does Pilotwave+ support multi-cluster management?

▼

Answer: Multi-cluster management is not currently supported in the current version of Pilotwave+. However, this feature is planned for future releases as part of our enterprise roadmap. Each Pilotwave+ instance currently manages a single Kubernetes cluster, but you can deploy multiple instances for managing multiple clusters independently.

6. Can Pilotwave+ support private Helm repository resources?

▼

Answer: Yes, Pilotwave+ supports private Helm repositories. When adding a new repository, you can access advanced settings to configure authentication credentials. Currently, we support HTTPS-based authentication methods. You can securely add private repositories by providing the necessary authentication tokens or credentials in the repository configuration interface.

7. What preparations are needed before downloading Pilotwave+? Do I need to pre-install Istio?

▼

Answer: No preparation is required beyond having a basic Kubernetes cluster. You do not need to pre-install Istio or any other components. Pilotwave+ can be deployed on a clean Kubernetes cluster and includes functionality to install and configure Istio, service mesh components, and other required resources directly through the platform interface.

8. Is authentication credential storage secure?

▼

Answer: Yes, Pilotwave+ implements secure authentication storage practices. Basic authentication credentials are stored using hashed passwords in environment variables, not as plain text. User credentials are properly encrypted and follow security best practices. The system uses industry-standard hashing algorithms and secure storage mechanisms to protect sensitive authentication information.

9. Which Kubernetes distributions are supported? Will it work with cloud provider managed Kubernetes services?

▼

Answer: Pilotwave+ has been primarily tested and validated on official kubeadm-created clusters. Other Kubernetes distributions may have limited support, though basic functionality should work across most standard Kubernetes environments. Cloud provider managed services (EKS, GKE, AKS) may work but haven't been extensively validated. We recommend testing in a development environment first when using non-kubeadm distributions.
